# Bloom Video Editor 機能実装リスト

作成日: 2026-06-01
更新日: 2026-06-03

## 目的

このファイルは、Bloom Video Editorプロトタイプの機能を開発管理するための一覧です。

READMEは使い方、`outputs/project-handoff.md`は引き継ぎ、ここでは「何が実装済みで、次に何を作るか」を管理します。

## 状態の定義

| 状態 | 意味 |
| --- | --- |
| 実装済み | 現在の静的プロトタイプ上で操作でき、基本動作を確認済み |
| 一部実装 | UIまたは一部ロジックはあるが、完成条件には不足がある |
| 未実装 | まだUIまたはロジックがない |
| 保留 | 今すぐ作らず、技術選定や運用方針を決めてから進める |

## 優先度の定義

| 優先度 | 意味 |
| --- | --- |
| Must | MVP、最小実用版に必要 |
| Should | 使い心地を上げるため早めに必要 |
| Could | 差別化や拡張として有効 |
| Won't | 現フェーズではやらない |

## 現在の実装サマリー

2026-06-03時点では、静的HTMLプロトタイプとして以下まで進んでいます。

- ホーム、編集、テンプレート画面の切り替え
- 写真ファイル読み込み、簡易補正、比率変更、PNG保存
- 動画ファイル読み込み、プレビュー、トリム範囲指定、範囲書き出しUI
- ローカルffmpeg補助による動画範囲書き出し
- 操作履歴、もどす、やり直す
- 「ここまで覚える」によるルーチン保存
- ホーム画面の「わたしのルーチン」からルーチン再利用
- 編集画面での保存済みルーチン削除
- PWA用のmanifestとservice worker

## 優先実装順

次に進めるなら、以下の順がおすすめです。

1. プロジェクト作成をlocalStorageで実データ化する
2. ホームの最近のプロジェクトを保存データから描画する
3. 写真/動画読み込み結果をプロジェクトに紐づける
4. トリム設定と写真補正設定を保存する
5. タイムラインのクリップ配列を実装する
6. テキストレイヤーを追加する
7. BGM読み込みと音量調整を追加する
8. 書き出し設定をプロジェクト単位で保存する
9. テンプレート選択をプロジェクト作成に接続する
10. PWAをスマホ実機で確認する

## 次に実装するおすすめ

次は「プロジェクト作成」と「最近のプロジェクト一覧」を実データ化するのがよいです。

理由:

- 現在のホーム画面を大きく変えずに、本物のアプリらしい状態へ進められる。
- 既に写真/動画読み込み、ルーチン、書き出しUIがあるため、それらをプロジェクトデータに載せられる。
- ルーチンだけがlocalStorage保存されている状態から、プロジェクトも保存される状態へ自然に拡張できる。

完了条件:

- 「動画を作る」または「写真を編集」から新しいプロジェクトが作成される。
- プロジェクト名、作成日時、更新日時、編集モードが保存される。
- ホームの最近のプロジェクトがlocalStorageから描画される。
- 最近のプロジェクトを押すと、該当プロジェクトとして編集画面を開ける。

## 技術検討メモ

- 動画素材や音声素材そのものはGitに入れない。
- ブラウザ版では、素材ファイルの永続保存に制限がある。
- ローカル検証ではObject URLとlocalStorageで進め、正式化時にIndexedDBやFile System Access APIを検討する。
- 動画書き出しは、ブラウザ内処理、ローカルffmpeg補助、将来のネイティブ/サーバー処理を分けて考える。
- iPhone/Androidアプリ化する場合は、React Native / Expoまたはネイティブ実装を別途検討する。
