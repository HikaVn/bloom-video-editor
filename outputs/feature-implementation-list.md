# Bloom Video Editor 実装機能一覧

作成日: 2026-06-01

## 前提

現在は `work/mobile-ui-prototype` に、ホーム / 編集 / エフェクト / 書き出し / ここまで覚える の静的プロトタイプがある。

次の実装では、見た目の完成度を保ちながら「実際に動画編集アプリとして使える状態」へ段階的に進める。

## フェーズ1: MVPで必須

### 1. プロジェクト作成

目的: ユーザーが新しい動画編集を開始できるようにする。

実装内容:

- 新規作成ボタン
- プロジェクト名の自動生成
- プロジェクト一覧への追加
- 最終更新日時の保存
- 下書きとして自動保存

完了条件:

- ホームから新規作成して編集画面へ移動できる
- 作成したプロジェクトがホームの一覧に残る

### 2. メディア読み込み

目的: 端末内の動画 / 画像 / 音声を編集素材として追加できるようにする。

実装内容:

- 動画ファイル選択
- 画像ファイル選択
- 音声ファイル選択
- 読み込み中ステータス
- 対応形式チェック
- 大きすぎる素材の警告

完了条件:

- 端末から選んだ素材がタイムラインに追加される
- 未対応ファイルを選んだ時に分かりやすく案内される

### 3. プレビュー再生

目的: 編集結果をその場で確認できるようにする。

実装内容:

- 再生 / 停止
- 再生位置の表示
- サムネイル表示
- プレビュー画面へのフィルター反映
- 音声の再生

完了条件:

- タイムライン上の素材を再生できる
- フィルターやテキストがプレビューに反映される

### 4. タイムライン編集

目的: 基本的な動画編集を成立させる。

実装内容:

- クリップ追加
- クリップ選択
- 分割
- 削除
- 並び替え
- 再生ヘッド移動
- もどす / やり直す

完了条件:

- 複数クリップを並べて、分割と削除ができる
- 操作後にプレビューと履歴が更新される

### 5. テキスト追加

目的: Vlogや日常動画に必要なタイトル / 字幕を入れられるようにする。

実装内容:

- テキスト追加
- 文字編集
- 位置調整
- 表示時間調整
- タイトルテンプレート適用

完了条件:

- 編集画面からテキストを追加し、プレビューで確認できる

### 6. フィルター適用

目的: 現在のエフェクト画面を実機能化する。

実装内容:

- ピュアピンク
- ウォーム
- ラベンダー
- ミルキーベージュ
- フィルター強度
- 選択中状態の保存

完了条件:

- フィルター選択がプレビューとプロジェクトデータに反映される

### 7. スタンプ追加

目的: かわいい素材をワンタップで動画に載せられるようにする。

実装内容:

- スタンプ選択
- 位置調整
- サイズ変更
- 回転
- 表示時間調整

完了条件:

- スタンプを追加して、プレビュー上で動かせる

### 8. 音楽 / BGM

目的: 動画にBGMや録音を追加できるようにする。

実装内容:

- BGMファイル追加
- 音量調整
- ミュート
- フェードイン / フェードアウト
- 音声トラック表示

完了条件:

- BGM付き動画をプレビュー再生できる

### 9. 書き出し

目的: 編集結果を動画ファイルとして保存できるようにする。

実装内容:

- 720p / 1080p / 4K選択
- MP4書き出し
- 書き出し進捗
- キャンセル
- 完了表示
- 保存先表示

完了条件:

- 編集した動画をMP4として保存できる

### 10. ここまで覚える

目的: 複数操作をルーチンとして保存し、次回ワンタップで再利用できるようにする。

実装内容:

- 操作履歴の記録
- ルーチン名入力
- ルーチン保存
- 保存済みルーチン一覧
- ルーチン再適用
- 最大保存数の制御

完了条件:

- フィルター / テキスト / スタンプなどの操作をまとめて保存し、別プロジェクトに再適用できる

## 最初の開発スプリント案

### Sprint 1

- プロジェクトデータ構造を作る
- 新規作成を実データ化する
- ホームのプロジェクト一覧を localStorage から描画する
- 既存UIの静的カードを実データに置き換える

### Sprint 2

- ファイル選択で動画 / 画像を追加する
- タイムラインにクリップを表示する
- プレビューに選択素材を表示する

### Sprint 3

- フィルターを実データに保存する
- テキスト追加を実装する
- 操作履歴をプロジェクト単位で保存する

### Sprint 4

- 書き出し設定をデータ化する
- ブラウザ版ではまず設定保存まで実装する
- ネイティブ / サーバー / FFmpeg の書き出し方式を決める

## 技術検討が必要な箇所

- 動画書き出しをブラウザ内で行うか、ネイティブ側で行うか
- FFmpeg.wasm を使うか、サーバー / デスクトップ処理に寄せるか
- iOS / Android を React Native / Expo で作るか
- 美容補正に MediaPipe / WebGL / ネイティブSDK を使うか
- 素材ファイルをローカル保存するか、クラウド保存するか

## 次に実装するおすすめ

最初は「プロジェクト作成」と「localStorageによるプロジェクト一覧保存」から始める。

理由:

- 現在のUIを大きく壊さず実機能化できる
- ホーム画面のカードが本物のデータになる
- その後のメディア読み込み、ルーチン、書き出し設定を同じデータ構造に載せられる
